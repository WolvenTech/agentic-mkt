# ADR-003: Gemini 2.5 Flash as Worker LLM

## Status

Superseded — the worker LLM is now **OpenAI `gpt-4.1-mini`** across all agent configs (see [`src/call-agent/logic.ts`](../src/call-agent/logic.ts) `DEFAULT_PROVIDER`/`DEFAULT_MODEL`). No follow-up ADR was written when the provider changed; this record is kept for the original alternatives-considered rationale (why Gemini was chosen over Claude at the time, and why the schema was built provider-agnostic).

## Date

2026-06-22

## Context

[ADR-001](adr-001.md) specified Claude Sonnet 4.6 via the Anthropic Messages API as the default worker model for `linkedin-writer`. During design, the stakeholder chose to use the n8n **Google Gemini node** with **gemini-2.5-flash** instead, replacing Claude for the initial green run.

Constraints: sub-60-second draft delivery target; structured JSON output `{ deliverable_markdown, resumo, autochecagem }`; n8n already hosts Google AI credentials; human review remains the quality gate.

## Decision

Use **Google Gemini 2.5 Flash** via the n8n **Gemini node** as the initial worker LLM for `linkedin-writer`:

- Provider field in `agents/linkedin-writer.json`: `"provider": "google"`, `"model": "gemini-2.5-flash"`
- Call Agent sub-workflow routes to the Gemini node based on agent JSON `provider` value
- System prompt instructs JSON-only response matching the output schema; Code node parses and validates required keys
- Cost target (< $0.05/draft) remains informational; Flash model supports lower latency and cost vs Sonnet

Claude Sonnet 4.6 remains a documented future option once provider-agnostic routing is validated.

## Alternatives Considered

### Alternative 1: Claude Sonnet 4.6 (originally specified default)

- **Description**: Anthropic Chat node or HTTP Messages API with Claude Sonnet 4.6.
- **Pros**: Matches the original spec; strong instruction-following for brand voice; Wolven may already have Anthropic billing.
- **Cons**: Requires Anthropic credential setup in n8n if not present; slightly higher cost per draft.
- **Why rejected**: Stakeholder chose Gemini for the initial green run.

### Alternative 2: Provider-agnostic with Claude default

- **Description**: Agent JSON specifies provider; ship Claude as default, Gemini as alternate.
- **Pros**: Future-proof; easy A/B between providers.
- **Cons**: Two credential paths to test before the initial green run; adds routing complexity.
- **Why rejected**: Stakeholder wanted Gemini-only initially; the `provider` field in JSON preserves the swap path (which was later exercised to move to OpenAI).

### Alternative 3: Gemini 2.5 Pro

- **Description**: Higher-quality Gemini model for draft quality.
- **Pros**: Better prose quality; still single Google credential.
- **Cons**: Higher latency and cost; may miss 60s target under load.
- **Why rejected**: Stakeholder selected gemini-2.5-flash for speed/cost.

## Consequences

### Positive

- Leveraged existing n8n Google AI / Gemini node integration at the time.
- Flash model aligned with initial latency and cost targets.
- Agent JSON `provider` + `model` fields enabled a later provider swap (to OpenAI) without workflow restructure — this is the mechanism that made the eventual Gemini → OpenAI move low-risk.

### Negative

- The originally specified Claude default was superseded for the initial implementation.
- Gemini JSON adherence required stronger prompt engineering or post-parse validation vs Anthropic/OpenAI structured outputs.

### Risks

- **Malformed JSON from the model** — Mitigation: Code node validates keys; on failure, post error comment on task and leave in-progress or move to a blocked state with diagnostic. (This validation path is provider-agnostic and remains in place after the OpenAI swap.)
- **Brand voice quality variance** — Mitigation: human review gate; acceptance criteria field guides output.

## Implementation Notes

- Temperature and max tokens set conservatively in agent JSON (defaults: temperature 0.7, max_output_tokens 1024).
- The `provider`/`model` fields being data (not code) is what let the project later swap to OpenAI `gpt-4.1-mini` without a workflow rewrite — see the current values in `agents/*.json`.
